# Documentation

## Rules

- The project root `README.md` is the summary document for the project.
- All detailed documentation lives under `docs/`.
- Documentation files must always use the `.md` extension.
- Keep all information that matters, but consolidate duplicated content into a single canonical document.
- When documentation moves, update every in-repo reference to the new location.
